## Project

KeysMirror is a macOS 13+ menu-bar app that maps keyboard shortcuts (and mouse side-buttons) to mouse-click events at fixed positions inside a target application's window — without moving the cursor. Swift 5.9 / Swift 6.3 toolchain, AppKit + SwiftUI, no third-party dependencies.

## Build, run, test

The Xcode project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml` (which is `.gitignore`d but checked in). Regenerate after editing `project.yml`:

```bash
xcodegen generate
```

The fastest dev loop is `./build.sh` — it builds with `xcodebuild` to `~/KeysMirror_build`, ad-hoc codesigns (required so AX permission persists across builds), kills any running instance, and `open`s the new app.

```bash
./build.sh           # Debug
./build.sh Release
```

Run a single test from the command line:

```bash
xcodebuild test -project KeysMirror.xcodeproj -scheme KeysMirror \
  -destination 'platform=macOS' \
  -only-testing:KeysMirrorTests/CoordinateConverterTests/testAbsolutePoint
```

Tail the runtime log:

```bash
tail -f ~/Library/Caches/KeysMirror/keysmirror.log
# or via Console.app / `log stream --predicate 'subsystem == "com.keysmirror.KeysMirror"'`
```

User mappings live at `~/Library/Application Support/KeysMirror/mappings.json`.

## Architecture

The app is a single-window-controller AppKit app driven by `AppDelegate` (`KeysMirror/App/`). Almost everything is `@MainActor` and exposed as a `.shared` singleton — there is no DI container.

### Event flow (the hot path)

1. **`KeyInterceptor`** installs a `CGEventTap` at `.cgSessionEventTap` for `keyDown`, `rightMouseDown`, and `otherMouseDown`. The C callback is `nonisolated static` and hops onto the main actor via `MainActor.assumeIsolated`.
2. For each event it queries `MappingStore.enabledProfile(bundleIdentifier:)` against `NSWorkspace.frontmostApplication`.
3. Before matching keyboard events, it calls **`WindowLocator.isFocusedElementTextInput(for:)`** — if the AX focused element is `AXTextField` / `AXTextArea` / `AXComboBox` / `AXSearchField`, the key is passed through untouched (text-input protection). **Do not add `AXWebArea` to that role set** — iOS-on-Mac games expose their entire render surface as `AXWebArea`, and adding it silently disables all in-game mappings.
4. On match, **`WindowLocator.focusedWindowFrame(for:)`** reads the target window's AX frame, **`CoordinateConverter.absolutePoint`** turns the stored relative offset into screen coords, and **`ClickSimulator.leftClick`** dispatches the click.
5. If `mapping.blockInput` is true, the original key is swallowed (`return nil`); otherwise it's passed through.

### Click delivery — two strategies

`ClickSimulator` branches on whether the target is a native macOS app or iOS-on-Mac (detected via `LSRequiresIPhoneOS` in the target's `Info.plist`):

- **Native:** `CGEvent.postToPid(pid)` — bypasses Window Server entirely, the cursor never moves, no cursor management needed.
- **iOS-on-Mac:** `event.post(tap: .cgSessionEventTap)` — Window Server *will* move the cursor to `mouseCursorPosition`, so the call is wrapped in `CGAssociateMouseAndMouseCursorPosition(0)` + `CGWarpMouseCursorPosition(savedPos)` to freeze and restore cursor position.

Chromium browsers are quirky in the same area — if you change click delivery, sanity-check Edge/Chrome before shipping.

### Tap-disabled and sleep/wake recovery

Two recovery paths converge on `KeyInterceptor.start()`:

- **Tap disabled by system:** When the kernel sends `tapDisabledByTimeout` / `tapDisabledByUserInput`, the underlying `CGEvent` C pointer is **null**. Swift bridges it as non-Optional, so the callback explicitly rewraps it as `Optional` (see `UnsafeOptionalEvent` in `KeyInterceptor.swift`) — calling `Unmanaged.passRetained` on the bridged event would crash. `TriggerRecorder` and `PointRecorder` follow the same pattern; mirror it for any new `CGEventTap`.
- **Sleep/wake:** `AppDelegate` observes `NSWorkspace.screensDidWakeNotification` and `NSWorkspace.didWakeNotification` and re-runs `keyInterceptor.start()` (which calls `_teardown()` first to clear stale CFMachPort/RunLoopSource state).

### Coordinate systems

`CoordinateConverter` is the single source of truth for AX (top-left origin, Y-down) ↔ AppKit (bottom-left origin, Y-up) conversions. Mappings are stored as window-relative offsets in **AX coordinates**; the overlay view converts to AppKit coords via `axScreenPointToAppKit` before positioning the `NSPanel`.

### UI / overlay

- `StatusBarController` owns the menu-bar icon (the green flash on hit comes from `flashActivity()`).
- `ConfigurationWindow` + `MappingEditorView` + `AppPickerView` are SwiftUI hosted in an `NSWindowController`.
- `OverlayController` runs a 0.5 Hz timer that draws translucent dots over each mapping's screen position. It caches `(bundleId, frame, profileVersion)` so most ticks skip the AX IPC entirely — when changing it, preserve that cache or you'll bring back the per-tick AX call that used to spike CPU.
- `TriggerRecorder` and `PointRecorder` install their own short-lived `CGEventTap`s during recording — they share the null-event handling pattern from `KeyInterceptor`.

### Data model

`AppProfile` (one per target bundle ID) → many `KeyMapping`. Persisted by `MappingStore` as JSON to Application Support. Profiles match by **case-insensitive** bundle ID (`enabledProfile`).

## Permissions and entitlements

The app requires only **Accessibility** (`AXIsProcessTrusted`). It is **not sandboxed** (`ENABLE_APP_SANDBOX: NO`, `ENABLE_HARDENED_RUNTIME: NO`) — required for `CGEventTap` and `postToPid`. The entitlements file is intentionally minimal; do not add sandbox or hardened-runtime entitlements without re-validating event-tap behavior.

Because the app is ad-hoc-signed (`CODE_SIGN_IDENTITY: "-"`), **rebuilding invalidates the previously granted Accessibility permission unless you preserve the binary identity** — `build.sh` re-signs in place to keep the same code signature, which is why the system remembers the grant. If you switch to a different signing identity or path, expect to re-grant in System Settings.

## Conventions

- Comments and log strings in this codebase are in **Simplified Chinese**; match that style when editing existing files. New top-level docs (READMEs, this file) can be English.
- All UI/event code is `@MainActor`. CG callbacks must round-trip through `MainActor.assumeIsolated`.
- Logging goes through `AppLogger.shared.log(_:type:)` with type tags `INFO` / `WARN` / `ERROR` / `TRACE` / `ACTION`. The in-UI log panel reads from `AppLogger.logs` (200-entry ring buffer); persistent log is at `~/Library/Caches/KeysMirror/keysmirror.log` and is **truncated on every launch**.
